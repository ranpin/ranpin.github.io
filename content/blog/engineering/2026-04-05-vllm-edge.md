vLLM 是主流的大模型推理框架,但要跑在车机这类端侧设备上,需要针对性适配。

## 为什么要适配

- 端侧算力/显存有限,必须依赖**量化**
- 后端不是 CUDA,而是高通的 **QNN**
- 对**首 token 延迟**和**稳定性**要求高于吞吐

## QNN 后端接入

把推理后端从默认切到 QNN,核心是实现一个 executor 适配层:

```python
# 简化示意:将 vLLM 的推理调用委托给 QNN runtime
class QnnExecutor:
    def __init__(self, model_path: str):
        self.session = qnn.Session(model_path, backend="htp")  # HTP = 高通 NPU

    def execute(self, input_ids, sampling_params):
        logits = self.session.run({"input_ids": input_ids})
        return sample(logits, top_p=sampling_params.top_p)
```

## 调优清单

1. 开启 **paged attention**,降低长上下文的显存碎片
2. 用 **AWQ** 量化权重,激活值保持 FP16
3. 合理设置 `max_num_seqs`,避免车机内存抖动
4. 预热(warm-up)一次,规避首次推理的冷启动尖峰

> 端侧的目标不是"最高吞吐",而是"在受限资源下稳定达标"。
